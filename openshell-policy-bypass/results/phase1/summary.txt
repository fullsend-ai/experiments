Key Finding: Proxy Binary Tracking Mechanism
=============================================
The proxy checks binary identity at TCP connection time using /proc/<pid>/exe,
then walks the parent process tree via /proc/<pid>/status PPid chain.

The OPA policy (sandbox-policy.rego) has three matching modes:
  1. Direct match:   /proc/<pid>/exe matches a policy binary path
  2. Ancestor match: a parent process in the PPid chain matches a policy binary
  3. Glob match:     binary path matches a glob pattern in the policy

This explains the results:
  - Shebang: /proc/pid/exe = /usr/bin/python3 (interpreter). No safe-push ancestor.
  - Exec:    /proc/pid/exe = /usr/bin/git (replaced). No safe-push ancestor.
  - Subprocess: /proc/pid/exe = git-remote-http, but safe-push is in PPid chain. Ancestor match.
  - HTTP:    /proc/pid/exe = /usr/local/bin/safe-push. Direct match.
